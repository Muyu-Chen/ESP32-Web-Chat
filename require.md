# ESP32 聊天服务器 Demo — 需求文档 v1

## 1. 系统目标

* 在 **无互联网环境** 下，ESP32 启动后创建 Wi-Fi 热点（SoftAP）。
* 用户手机/电脑连上后，会被 **自动重定向到聊天室网页**（Captive Portal 效果）。
* 提供一个 **轻量聊天室**，支持群聊和私聊（广播发送，客户端过滤展示），消息仅为 **文字**。
* 服务器仅保留最近 **100 条消息**（FIFO 缓存），不做长期存储，掉线可补齐。

---

## 2. 功能需求

### 2.1 网络与接入

* **热点服务**

  * ESP32 启动时创建 SoftAP（默认 SSID `ESPChat`，可设 WPA2 密码）。
  * 最大支持 5\~8 个客户端。

* **Captive Portal 自动弹窗**

  * 配置 DNS 劫持：所有域名请求都解析到 `192.168.4.1`。
  * HTTP 服务器对所有请求返回聊天室页面。
  * 支持 **“登录到网络”** 协议，让手机/电脑自动弹出浏览器进入聊天室。

---

### 2.2 聊天功能

* **WebSocket 通道**

  * 路径 `/ws`。
  * 用户消息通过 WebSocket 发送到 ESP32。
  * ESP32 将收到的消息立即广播给所有在线用户。

* **消息模型（JSON）**

  ```json
  {
    "type": "text",
    "from": "uuid-123",
    "to": {
      "all": true,
      "users": ["uuid-456", "uuid-789"]
    },
    "name": "Alice",
    "data": "你好！",
    "id": 42,
    "timestamp": 1694601234
  }
  ```

  * `to.all = true` → 群聊消息。
  * `to.users` → 多个私聊对象 ID。
  * 消息依然广播给所有人，但客户端只显示与自己相关的部分。

* **在线用户查询（新增）**

  * 客户端发送消息类型 `getOnlineUser`：
    ```json
    {
      "type": "getOnlineUser",
      "from": "uuid-123"
      // to, name, data 可省略
    }
    ```
  * 服务端返回当前在线用户列表（广播给所有人）：
    ```json
    {
      "type": "onlineUsers",
      "from": "server",
      "to": { "all": true, "users": ["uuid-123", "uuid-456", "uuid-789"] },
      "data": [
        {"id": "uuid-123", "name": "Alice"},
        {"id": "uuid-456", "name": "Bob"},
        {"id": "uuid-789", "name": "Carol"}
      ],
      "timestamp": 1694601234
    }
    ```

* **群组创建消息（新增）**

  * 客户端拉群时，发送 `newGroup` 类型消息：
    ```json
    {
      "type": "newGroup",
      "from": "uuid-123",
      "name": "Alice",
      "groupId": "group-xxx",
      "groupName": "Alice, Bob, Carol",
      "to": {
        "all": true,
        "users": ["uuid-456", "uuid-789"]
      },
      "data": "Alice 创建了群聊",
      "timestamp": 1694601234,
      "id": 43
    }
    ```
  * 服务端确认后广播给所有人，客户端用此消息生成新的群聊记录。

* **消息存储（FIFO 缓存）**

  * ESP32 仅保存最近 **100 条消息**（JSON 环形缓存）。
  * 新消息覆盖最旧消息。
  * ESP32 重启后所有记录清空。

* **掉线重连补偿**

  * 客户端重新连接后，请求 `since_id = last_seen_id` 之后的消息，最多拉取 100 条。

---

### 2.3 用户识别与昵称

* **用户 ID**

  * 客户端首次进入时生成一个 UUID，存入 `localStorage`。
  * 同一设备重复进入时，沿用相同 UUID。

* **昵称**

  * 用户进入聊天室时可自定义昵称（默认值如 `User123`）。
  * 昵称显示在消息上。

---

### 2.4 前端 UI

* **聊天室界面**

  * **上方**：消息滚动区域（显示群聊/私聊消息）。
  * **下方**：输入框 + 发送按钮。
  * 系统提示消息（用户加入/离开）以特殊样式展示。

* **二级菜单结构（新增）**

  * **第一级菜单**：用户首页
    * 展示自己的用户 ID（UUID）和昵称
    * "进入聊天" 按钮进入第二级菜单
    * 显示当前连接状态

  * **第二级菜单**：聊天列表页面
    * **群聊入口**（第一行，永远置顶）
      * 显示 "全体群聊" / "Group Chat"
      * 显示最后一条消息的预览
      * 点击进入全体群聊页面
    * **在线用户列表**
      * 根据 `onlineUsers` 消息动态显示当前在线的人（不包括自己）
      * 显示用户昵称，点击进入与该用户的私聊页面
    * **聊天历史列表**
      * 显示之前聊过天的人/群（从服务端消息记录中统计）
      * 按最后一条消息时间排序
    * **右上角 + 按钮**：创建群聊
      * 点击后进入群聊选择页面
      * 展示所有在线用户（除自己外），多选模式
      * 选择完毕后点击 "创建群聊" 发送 `newGroup` 消息
      * 返回聊天列表后，新群聊出现在列表中

* **消息显示规则**

  * 群聊：所有人都能看到。
  * 私聊：消息广播给所有人，但客户端只显示 `to.users` 中包含自己的消息。
  * 群聊消息：只显示 `to.users` 中包含自己的消息（新群组）。

## 2.5 前端实现方案（新增）

## 2.5 前端实现方案（新增）

* **最小化改动策略**

  * 保留原有聊天界面（消息显示 + 输入框）
  * 在聊天界面顶部添加：
    - **对话标题**：显示当前对话名称（"全体群聊"、"私聊 - Bob" 或 "群聊 - Alice, Bob"）
    - **菜单按钮**（左上）：打开对话列表
  
* **对话列表界面（简化版）**

  * **全体群聊**（第一项，固定）：点击进入全体群聊
  * **在线用户列表**：显示当前在线的人（除自己外）
    - 点击进入与该用户的私聊
  * **群聊创建**（右上 + 按钮）
    - 弹出对话框，勾选多个在线用户
    - 点击 "创建群聊" 发送 `newGroup` 消息
  * **历史对话**：自动从消息记录中统计，显示在下方

* **数据结构（极简化）**

  * **currentConversation**：当前选中的对话 ID
    ```javascript
    {
      "id": "global" | "uuid-456" | "group-xxx",
      "name": "全体群聊" | "Bob" | "群聊名称",
      "members": ["uuid-123", "uuid-456", ...]  // 仅群聊需要
    }
    ```
  * **conversations**：对话快速索引（localStorage）
    ```javascript
    {
      "global": { "name": "全体群聊", "type": "global" },
      "uuid-456": { "name": "Bob", "type": "private" },
      "group-xxx": { "name": "Alice, Bob", "type": "group", "members": [...] }
    }
    ```

* **核心逻辑（保持简单）**

  1. **默认进入全体群聊**（`currentConversation.id = "global"`）
  2. **切换对话时**：更改 `currentConversation.id`，前端自动重新渲染现有消息（过滤显示）
  3. **接收消息后**：
     - 保存到 `esp-chat-messages`（全量保存，不分对话）
     - 检查是否属于当前对话，是则显示
     - 自动更新 `conversations` 中的对话元数据
  4. **发送消息时**：
     - 根据当前 `currentConversation` 设置 `to` 字段
     - 全体群聊：`to.all = true`
     - 私聊：`to.users = [targetUserId]`
     - 群聊：`to.users = [members...]`

* **在线用户获取**

  * 连接成功后发送 `getOnlineUser` 消息
  * 服务端返回 `onlineUsers` 消息（包含所有在线用户列表）
  * 前端更新对话列表中的在线用户部分

## 3. 非功能需求

* **实时性**：文字消息延迟 < 300ms。
* **容量**：消息缓存上限 100 条。
* **可靠性**：掉线重连能补齐最近消息。
* **易用性**：用户无需安装 App，自动弹出网页即可聊天。
* **存储**：不做长期保存，ESP32 重启清空。

---

## 4. 不包含的功能

* 图片/视频/文件传输。
* 用户登录认证。
* 长期存储/历史互助补偿。
* 消息加密
